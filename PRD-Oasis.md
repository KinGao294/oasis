## 一、产品概述

### 1.1 品牌名称

**Oasis**（绿洲）

> 在信息的荒漠中，这里是你的绿洲。

### 1.2 产品愿景

> **在信息过载的时代，你的注意力流向哪里，你就会成为什么样的人。**

我们相信：真正的自由不是拥有无限的选择，而是能够专注于真正重要的事物。

当算法用无穷无尽的信息流填满你的每一秒，当「刷完这条再走」变成每晚的自我欺骗，当你在信息的海洋里越来越焦虑却越来越空虚——

**Oasis 要做的，是成为你在信息荒漠中的那片绿洲。**

### 1.3 产品定位

**Oasis** 不是又一个信息聚合工具，而是你的「**信息绿洲**」——

一个经过你亲手筛选、只留下值得深耕的高质量信息源的空间。在这里：
- 没有算法投喂，只有你主动选择关注的人
- 没有无限下拉，只有今天值得你花时间的内容
- 没有「可能感兴趣」，只有「确实让我成长」

**一句话定位**：为终身学习者打造的信息绿洲，让你在最想成长的领域持续深耕。

**品牌 Slogan**：
- Your oasis in the information desert.
- 在信息的荒漠里，找到你的绿洲。

### 1.3 我们要解决的真正问题

**表面问题**是信息分散、效率低下。

**深层问题**是：在注意力被无限争夺的时代，我们正在失去「深度」的能力。

| 现象 | 本质 | Oasis 的回答 |
|------|------|----------------|
| 每天刷 3 小时手机，却不记得看了什么 | 被动消费，没有目的 | 你决定看什么，而不是算法决定 |
| 关注了 100 个博主，真正看的不到 10 个 | 订阅焦虑，害怕错过 | 精选信息源，Less is More |
| 看完视频觉得有用，但从不行动 | 浅尝辄止，缺乏系统 | 按领域深耕，形成知识体系 |
| 想找之前看过的观点，却怎么也找不到 | 信息流过即忘 | 全文可搜索，变成你的知识库 |

**具体痛点场景**：

| 痛点 | 现状 | Oasis 方案 |
|------|------|--------------|
| 信息分散 | 需要在 YouTube、B站、X、播客 App 之间来回切换 | 一个页面看所有平台更新 |
| 错过更新 | 算法推荐导致错过关注博主的内容 | 基于订阅，不遗漏任何更新 |
| 时间浪费 | 打开视频才知道讲什么 | 先看文案摘要，再决定是否观看 |
| 无法检索 | 想找某个观点，不知道在哪个视频里 | 全文搜索所有视频文案 |

### 1.4 目标用户

**为谁而做**：那些相信「你吸收的信息塑造你是谁」的人。

```
用户画像：终身学习者

他们的特征：
- 对特定领域有强烈的学习欲望（AI、商业、创业、创作...）
- 已经有一份精心挑选的「信息源清单」
- 厌倦了算法推荐的垃圾信息，想要夺回注意力的主动权
- 希望把碎片时间用在真正有价值的内容上
- 相信持续输入高质量信息，会带来长期复利

他们的追求：
- 不是「知道更多」，而是「理解更深」
- 不是「刷得更快」，而是「留下更多」
- 不是「关注更多人」，而是「跟对的人学习」
```

### 1.5 核心价值主张

```

│                                                             │

│                                                             │
│   🔍  所有内容可搜索，你的第二大脑                            │
│                                                             │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.6 产品理念

**做减法，不做加法**
- 我们不会给你推荐「可能感兴趣」的内容
- 我们不会让你「无限下拉」
- 我们不会用红点和通知打扰你

**你的时间，你做主**
- 想看的时候打开，不想看的时候安静
- 每周一封摘要，够了

**深度优先于广度**
- 宁愿帮你深耕 5 个领域，也不帮你浅尝 50 个话题
- 完整的视频文案，让你真正消化而不只是划过

#### 交互细节

- 采用「药丸按钮」(Pill Button) 样式

- 选中状态使用品牌主色填充

- 筛选状态同步到 URL，支持分享

- 鼠标 hover 时轻微上浮 (translateY -2px) + 阴影加深

- 输入时实时显示匹配结果数量
- 搜索结果高亮匹配文字

**空状态**
- 无内容时显示友好提示："今天订阅的博主还没更新，去喝杯咖啡吧 ☕"
- 筛选无结果时："没有找到匹配的内容，试试调整筛选条件？"

## 七、用户体验细节

### 7.1 性能优化

- 图片懒加载 (Lazy Loading)
- 虚拟列表 (Virtual List) 处理大量内容
- 缩略图使用 WebP 格式，节省带宽
- 首屏优先加载，文案数据按需加载

### 7.2 无障碍 (A11y)

- 所有交互元素可键盘访问
- 图片有 alt 文本
- 颜色对比度符合 WCAG 2.1 AA 标准
- 支持屏幕阅读器

### 7.3 错误处理

| 场景 | 处理方式 |
|------|----------|
| 网络错误 | 显示重试按钮 + 友好提示 |
| 内容加载失败 | 显示占位图 + "加载失败" |
| 搜索无结果 | 显示空状态插图 + 建议 |
| 文案获取失败 | 显示"暂无文案"，不阻塞其他内容 |

### 7.4 快捷键

| 快捷键 | 功能 |
|--------|------|
| `⌘/Ctrl + K` | 打开搜索 |
| `/` | 聚焦搜索框 |
| `Esc` | 关闭详情/返回列表 |
| `J` / `K` | 上/下一条内容 |
| `O` / `Enter` | 打开详情 |
| `G` | 跳转原文 |

## 八、数据需求

### 8.1 核心数据实体

```typescript
// 博主/信息源
interface Source {
  id: string;
  name: string;
  avatar?: string;
  platform: 'youtube' | 'bilibili' | 'x' | 'podcast';
  domains: Domain[];
  feedUrl?: string;
  channelId?: string;  // YouTube
  uid?: string;        // Bilibili
  username?: string;   // X
}

// 领域
type Domain = 'AI' | '商业财经' | '出海全球化' | '自媒体创作';

// 内容项
interface FeedItem {
  id: string;
  sourceId: string;
  sourceName: string;
  sourceAvatar?: string;
  platform: Platform;
  domains: Domain[];
  
  // 内容
  title?: string;           // 视频标题 / null for tweets
  content?: string;         // 推文内容
  url: string;
  thumbnail?: string;       // 视频封面
  images?: string[];        // 推文图片
  duration?: number;        // 视频时长(秒)
  
  // 文案
  hasTranscript: boolean;
  transcriptPreview?: string;  // 前200字
  
  // 元数据
  publishedAt: string;      // ISO 8601
  createdAt: string;
}

// 完整文案
interface Transcript {
  feedItemId: string;
  source: 'youtube_caption' | 'bilibili_cc' | 'whisper';
  language: string;
  fullText: string;
  segments: TranscriptSegment[];
  wordCount: number;
}

interface TranscriptSegment {
  start: number;   // 秒
  end: number;
  text: string;
}
```

### 8.2 筛选与搜索

- 领域：多选

- 时间：单选（今天 / 近3天 / 近7天）

**搜索范围**
- 标题
- 博主名
- 文案全文（需要建立搜索索引）
